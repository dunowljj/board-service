# Decision Boundary Checklist

If ANY of the following is true → ADR required.

- A new domain concept is introduced
- A default behavior is added or changed
- Transaction boundary changes
- Persistence strategy changes
- Caching strategy changes
- Performance assumptions introduced
- Consistency model altered (eventual vs strong)
- API contract modified
- Authorization rules changed
- Data model altered (schema/index/invariant)

If uncertain → escalate.
